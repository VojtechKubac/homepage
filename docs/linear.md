# Linear (this repo)

Work for the portfolio site is tracked in Linear (team **Kubac**, workspace-specific).

## Homepage project for issues

**Assign every issue that ships or defines this site** (including portfolio content, project copy, KUA tickets for homepage work) to the Linear project **Homepage**.

Do not assign those issues to separate Linear projects that only represent a *portfolio entry* or external product (e.g. TCAE modules). Those extra Linear projects are optional buckets for roadmap or notes; the **issue** always stays on **Homepage** unless you explicitly choose otherwise.

## Creating issues via MCP

When using `save_issue`, set `project` to **`Homepage`** for homepage-related work. `team` is **`Kubac`**.
